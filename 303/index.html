연산자#

  • 연산자(Operator) : 산술 연산을 나타내는 기호
  • 피연산자(Operand) : 연산하는 대상

  • 피연산자 수에 따라

    • 단항 연산자(unary operator) : 하나의 피연산자만을 취함
    • 이항 연산자(binary operator) : 두나의 피연산자만을 취함
    • 삼항 연산자(ternary operator) : 세하나의 피연산자만을 취함

  • 표기법에 따라

    • 전위표기(prefix notation) :   +35   ++i
    • 후위표기(postfix notation) :   35+   i++
    • 중위표기(infix notation) :   3+5

  • 가환연산자(*, +, &, |, ^) 주의사항

    • 교환적이고 결합적이다.
    • 이 식의 평가 순서는 대수적으로 동치

    • 사용자가 괄호를 하여도 compiler가 임의로 재배치하여 평가하므로 side effect가 유발할 우려가 있다.

      • 임시 변수나 괄호를 사용하여 평가 순서를 명시적으로 나타내는게 바람직

 

산술연산자#

  • 가수 연산자 (Additive Operators) :  +  -#
    • 이항연산자
    • 오퍼랜드는 임의의 산술형태
    • Pointer에 대한 연산에도 쓰임
    • 결합순서는 --> 이다

 

  • 배수 연산자 (Multiplicative Operators) :  *  /  %(나머지)#
    • 이항연산자
    • *, /의 오퍼랜드는 산술형태

    • %의 오퍼랜드는 정수형태

      • 10.3 % 3.2 는 에러 --> fmod(10.3, 3.2) 함수 호출문을 사용

    • 정수/정수는 결과도 정수이다.

      • 5/4 --> 1,  5.0/4.0 --> 1.25
    • 결합순서는 --> 이다

   int a, b, c, d;

   a = 2;  b = -3;  c = 7;  d = -19;

  • 증감연산자 :  ++  --#
    • 단항연산자
    • 간결하고 이해가 쉽고 보다 효출적인 코드를 생성

    • 오퍼랜드는 좌변값(lvalue)을 갖는 산술형 데이터

      • 좌변값(lvalue)이란 대입문 좌측에 쓸 수 있는 것으로 l은 left 또는 load를 의미한다.
      • 즉, 저장 가능한 변수를 의미
      • 2 = a;  x + y = 10 처럼 상수나 수식을 대입문 좌측에 쓸 수 없다.

    • 부작용(side effect)에 주의해야 한다.

      • ex) x = 2;  y = i / ++x;  -->  y = 2 / 3 or 3 / 3  ?
      • 한 변수가 수식 내에 두번 이상 쓰이면 증감연산자 사용 금지
    • 결합순서는 <-- 이다.

    • 전위형(prefix)

      • 오퍼랜드를 먼저 1만큼 증가(감소) 시킨 후 수식에 참여
      1. a = b + ++i;   -->   i = i + 1;  a = b + i;

    • 후위형(Postfix)

      • 현재의 값으로 수식을 계산한 후 오퍼랜드는 1 증가(감소)
      1. a = b + i++;   -->   a = b + i;  i = i + 1;

 

  1. #include <iostream>
    using namespace std;

    void main(void)
    {
        // 예제 1
        int a = 1, b = 1;
        int aplus, bplus;

        aplus = a++;  // postfix : aplus = a, a = a + 1
        bplus = ++b;  // prefix : b = b + 1, bplus = b

        cout << "a aplus b bplus\n";
        cout << a << "   " << aplus << "   "
             << b << "   " << bplus << '\n';

        // 예제 2
        int count = 0, sum = 0;

        while (++count < 10)
            sum = sum + count;

        cout << count << "\t" << sum << "\n";

        // 예제 3
        count = 0, sum = 0;

        while (count++ < 10)
            sum = sum + count;

        cout << count << "\t" << sum << "\n";
    }

 

대입연산자(Assignment Operators)#

  • 대입연산자#
    • C에서 대입은 문이 아니고 식(연산)으로 취급한다.
    • 이항연산자
    • 왼쪽 오퍼랜드는 좌변값(lvalue)을 가져야 한다.
    • 오른쪽 오퍼랜드는 산술형태이다.
    • 결합순서는 <-- 이다.

 

  • 다중치환 (Multiple assignment)#
    • varN = varN_1 = ... = var1 = expr

    • good = 0;

      good = 0;

      better = good + 1;

Bit 연산자#

  • 오퍼랜드는 정수적형(integral)이다
  • 결합순서는 --> 이다. ( 단, ~는 <-- )
  • Bit 연산자 (bitwise logical operators) :  &   |   ^#
    • & : 비트곱 연산자 (bitwise AND)
    • | : 비트합 연산자 (bitwise inclusive OR)
    • ^ : 비트차 연산자 (bitwise exclusive OR)
  • 1의 보수 연산자 (Bitwise Complement) :  ~#
    • 연산결과는 오퍼랜드의 1의 보수가 된다.

  1. #include <iostream>
    using namespace std;

    // style이나 flag 지정에 비트연산이 자주 등장한다.
    #define STYLE_ONE        0x0001
    #define STYLE_TWO        0x0002
    #define STYLE_THREE        0x0004
    #define STYLE_FOUR        0x0008
    #define STYLE_FIVE        0x0010

    void main(void)
    {
        int myStyle = 0;

        myStyle = STYLE_ONE | STYLE_TWO;
        cout << "my style has style_one ? : " << (myStyle & STYLE_ONE) << endl;
        cout << "my style has style_three ? : " << (myStyle & STYLE_THREE) << endl;

        // add style
        myStyle |= STYLE_THREE;
        cout << "my style has style_three ? : " << (myStyle & STYLE_THREE) << endl;

        // remove style
        myStyle &= ~STYLE_ONE;
        cout << "my style has style_one ? : " << (myStyle & STYLE_ONE) << endl;
    }

 

  • Shift 연산자 :  <<   >>#
    • 우변 오퍼랜드에 의해 주어지는 값만큼 좌변 오퍼랜드를 좌측(<<) 또는 우측(>>)으로 bit 이동
    • E1 << E2 --> E1 * 2E2
